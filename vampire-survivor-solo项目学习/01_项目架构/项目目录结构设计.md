# 项目目录结构设计

## 目录结构概览

```
vampire-survivor-solo/
├── addons/           # 插件目录
├── assets/           # 美术素材（Sprite）、音频资源
├── Resources/        # Godot自定义资源文件（.tres）
│   ├── upgrades/     # 局内能力升级定义
│   ├── meta_upgrades/# 局外永久升级定义
│   └── theme/        # UI主题和字体
├── scenes/           # 游戏核心场景和脚本
│   ├── ability/      # 技能相关（Controller和Ability实体）
│   ├── autoload/     # 全局单例脚本
│   ├── component/    # 通用组件
│   ├── game_object/  # 游戏实体（玩家、敌人、经验球）
│   ├── manager/      # 游戏流程管理器
│   ├── UI/           # 用户界面
│   └── main/         # 游戏主循环入口
├── scripts/          # 脚本文件
└── project.godot     # 项目配置
```

## 设计原则

### 1. 功能模块分离

- **component/** - 通用组件，可挂载到任何实体
- **manager/** - 游戏流程控制器
- **ability/** - 技能系统专用逻辑
- **UI/** - 界面相关

### 2. 场景与脚本同名

```
scenes/component/health_component.gd  <->
scenes/component/health_component.tscn
```

### 3. Autoload 单例集中管理

```gdscript
# Autoload 脚本示例（game_events.gd）
extends Node
signal experience_viol_collected(number: float)
signal ability_upgrade_added(upgrade: AbilityUpgrade, current_upgrades: Dictionary)
```

## 最佳实践

1. **资源分类存放** - 美术资源放 assets，数据资源放 Resources
2. **组件复用** - 通用逻辑抽取为组件，避免重复代码
3. **脚本独立** - 脚本文件与场景文件分离，便于版本管理
4. **命名规范** - 使用 class_name 定义类名，方便类型检查
